# 정규 표현식 사용 현황 및 권장사항

## 🔍 현재 정규 표현식 사용 현황

### ✅ **현재 사용 중인 정규식들**
1. **주유소 패턴**
   ```regex
   /(.+?)\(상\)주/i     # (상)주 패턴 - 신뢰도 95%
   /(.+?)\(하\)주/i     # (하)주 패턴 - 신뢰도 95%
   /(GS칼텍스|SK엔크린|에쓰오일|현대오일뱅크|S-?OIL)셀프/i  # 셀프주유소
   ```

2. **편의점 패턴**
   ```regex
   /GS25/i                    # GS25
   /\bCU\b/i                  # CU (단어 경계)
   /(세븐일레븐|7-?ELEVEN)/i   # 세븐일레븐 다양한 표기
   ```

3. **온라인 서비스**
   ```regex
   /(GOOGLE|구글)/i           # 구글
   /(AMAZON|아마존|AWS)/i     # 아마존
   /(GODADDY|고대디)/i        # 고대디
   ```

## 💡 **정규 표현식 사용의 장단점**

### ✅ **장점**
1. **매우 빠른 처리 속도** - 메모리 내 패턴 매칭
2. **높은 정확도** - 특정 패턴에 대해 99% 이상 정확
3. **비용 효율성** - LLM 호출 대비 거의 무료
4. **예측 가능성** - 동일 입력에 동일 결과 보장

### ❌ **단점**
1. **유지보수 어려움** - 새 패턴 추가 시 코드 수정 필요
2. **제한된 유연성** - 예상하지 못한 변형에 취약
3. **복잡한 패턴 관리** - 정규식 자체의 복잡성
4. **확장성 제한** - 카테고리 추가 시 개발 필요

## 🎯 **권장사항**

### 1. **계층적 사용 전략** (현재 구현과 유사)
```
1순위: 정규식 (확실한 패턴) - 95% 신뢰도
2순위: 키워드 매칭 (일반적 패턴) - 70-90% 신뢰도  
3순위: LLM (미지의 패턴) - 변동적 신뢰도
```

### 2. **정규식 사용이 권장되는 케이스**
- **고정적 패턴**: `(상)주`, `(하)주` 같은 표준화된 패턴
- **높은 빈도**: 주유소, 편의점 등 자주 나타나는 거래
- **명확한 식별자**: 브랜드명, 고유 키워드

### 3. **정규식 대신 키워드를 사용할 케이스**
- **변형이 많은 패턴**: 카페, 음식점 등
- **지역별 차이**: 체인점이 아닌 개별 상호
- **신규 브랜드**: 정규식에 없는 새로운 패턴

### 4. **최적 조합 전략**
```
고신뢰도 정규식 (20% 처리) 
→ DB 키워드 매칭 (70% 처리)
→ LLM 추론 (10% 처리)
```

## 📊 **처리 속도 비교**
1. **정규식 엔진**: ~0.1ms (가장 빠름)
2. **CSV 정규식**: ~0.5ms (파일 I/O 포함)
3. **키워드 DB**: ~5-20ms (DB 쿼리)
4. **LLM 호출**: ~500-2000ms (가장 느림)

## 💰 **비용 비교**
1. **정규식**: 거의 무료
2. **키워드 DB**: DB 비용 (매우 저렴)
3. **LLM**: 호출당 비용 (상대적으로 비쌈)

## 🎯 **최종 권장 아키텍처**

```
[입력 거래 문자열]
        ↓
[1단계: 고신뢰도 정규식 엔진] ← 확실한 패턴 (주유소, 편의점 등)
        ↓ (매칭 실패 시)
[2단계: CSV 동적 정규식 엔진] ← 관리자가 추가한 패턴
        ↓ (매칭 실패 시)  
[3단계: DB 키워드 엔진] ← 학습된 키워드 룰
        ↓ (매칭 실패 시)
[4단계: LLM 추론] ← 완전히 새로운 패턴
        ↓
[5단계: 학습 루프] ← 피드백 반영 및 룰 생성
```

이 구조를 통해 **속도**, **정확도**, **비용 효율성**을 모두 달성할 수 있습니다.

## 개선 제안사항

1. **정규식 엔진 확장**: 음식점, 병원, 약국 등 추가 카테고리
2. **CSV 관리 도구**: 웹 UI를 통한 정규식 패턴 관리
3. **성능 모니터링**: 각 단계별 처리 시간 및 성공률 추적
4. **A/B 테스트**: 다양한 알고리즘 조합의 효과 측정
