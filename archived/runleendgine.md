# AI 기장 서비스 - 하이브리드 룰 엔진 작동 알고리즘 (v1.1)

**입력:** 사용자가 입력한 원본 거래 문자열 (예: "(주)스타벅스커피코리아 강남역점 9,800원")

---

### 🖥️ **1단계: 전처리 및 정규화 (Normalization)**

1.  **거래처명/금액 분리:** 문자열에서 **핵심 거래처명**("스타벅스커피코리아 강남역점")과 금액("9,800원")을 분리합니다.
2.  **불필요한 단어 제거:** 상호명에 흔히 포함되지만 분류에는 불필요한 단어들을 제거합니다.
    * 법인 형태: `(주)`, `주식회사`, `(유)`, `(합)` 등
    * 일반 명사: `지점`, `코리아`, `본점` 등
    * **결과:** "스타벅스커피" 라는 **`정제된 거래처명`**을 얻습니다.

---

### 🗂️ **2단계: 룰 DB 우선 검색 (Rule DB First)**

가장 빠르고 비용이 저렴한 DB 검색을 먼저 수행합니다. 이 단계의 목표는 **비싼 LLM 호출을 최소화**하는 것입니다. 우리가 함께 만들고 있는 **V4 키워드 룰 테이블**이 바로 이 `룰 DB`입니다.

1.  **완전/부분 일치 검색:** `정제된 거래처명`("스타벅스커피")이 `룰 DB`의 **'키워드'** 컬럼에 포함되어 있는지 검색합니다.
    * (성공) '스타벅스' 키워드와 일치하는 규칙을 찾았습니다. **➡️ 3단계로 이동**
    * (실패) 일치하는 키워드가 없습니다. **➡️ 4단계로 이동**

---

### 🤔 **3단계: 규칙 적용 및 사용자 상호작용 (Rule Application & User Interaction)**

`룰 DB`에서 규칙을 찾았을 때, 규칙의 내용에 따라 지능적으로 작동합니다.

1.  **신뢰도 확인:** 찾은 규칙의 **'신뢰도'** 점수를 확인합니다.
2.  **경우의 수 분기:**
    * **A. [신뢰도 높음 (90점 이상)]:** 질문이 불필요한 명확한 거래입니다.
        * 규칙에 정의된 **'선택지 1 결과 (태그)'** 와 **'(계정과목)'** 을 사용자에게 추천하거나 바로 확정합니다. (예: '택시' -> `#교통비`, `여비교통비`)
        * **➡️ 5단계로 이동**
    * **B. [신뢰도 중간 (90점 미만)]:** 사용자의 판단이 필요한 애매한 거래입니다.
        * 규칙에 정의된 **'질문 내용'**과 **'답변 선택지 1, 2'**를 사용자에게 버튼 형태로 제시합니다. (예: '음식점' -> "누구와 함께한 식사인가요? [직원 식사] [거래처 접대]")
        * 사용자가 선택한 답변에 연결된 **태그**와 **계정과목**으로 처리합니다.
        * **➡️ 5단계로 이동**

---

### 🧠 **4단계: LLM 호출 및 추론 (LLM Fallback)**

`룰 DB`에 정보가 없는, 처음 보는 거래처명을 처리하는 단계입니다.

1.  **LLM 호출:** **Gemini API**를 호출합니다.
2.  **지능적 프롬프트 구성:** LLM에게 효과적으로 질문합니다.
    > *"너는 한국 최고의 세무사 AI야. 사용자가 '**[정제된 거래처명]**'이라는 곳에서 돈을 썼어. 이 상호명으로 미루어 볼 때, 가장 가능성이 높은 업종과 추천 계정과목은 무엇이야? 두 번째 가능성도 있다면 알려줘."*
3.  **LLM 답변 분석:** Gemini가 생성한 답변(예: "커피 전문점으로 보이며, 복리후생비 또는 접대비로 처리할 수 있습니다.")을 분석하여 핵심 계정과목과 태그를 추출합니다.
4.  **사용자 확인:** LLM의 추론 결과를 사용자에게 추천 형식으로 제시하고 확인받습니다.
5.  **➡️ 5단계로 이동**

---

### ✨ **5단계: 학습 및 신뢰도 동적 업데이트 (Feedback & Dynamic Update)**

**이 부분이 대표님의 피드백을 반영하여 강화된 핵심 로직입니다.**

#### **(A) 긍정 피드백: 신뢰도 상승 📈**

* **A-1. [1순위 추천 확정 시]:** 사용자가 시스템의 첫 제안을 그대로 확정했습니다.
    * 해당 규칙의 **신뢰도를 +1점 상승**시킵니다. (최대 100점)
* **A-2. [질문 후 1순위 답변 선택 시]:** 시스템의 질문에 사용자가 1순위 답변을 선택했습니다.
    * 해당 규칙의 **신뢰도를 +1점 상승**시킵니다.

#### **(B) 부정 피드백: 신뢰도 하락 📉**

* **[질문 후 2순위 답변 선택 시]:** 사용자가 2순위 답변을 선택했습니다. 1순위 추천이 틀렸다는 의미입니다.
    * 해당 규칙의 **신뢰도를 -2점 하락**시킵니다. (최소 0점)
    * **(중요!)** 해당 규칙의 **1순위와 2순위의 순서를 서로 바꿉니다.** 다음 사용자에게는 더 정답에 가까웠던 선택지를 먼저 보여주게 됩니다.

#### **(C) 신규 규칙 생성 및 검증 🆕**

* **[LLM 추론 결과 확정 시]:** 4단계에서 LLM이 추론한 결과를 사용자가 최종 확정했습니다.
    * `**(정제된 거래처명, 확정된 태그, 확정된 계정과목)**` 조합을 **'신규 규칙 후보'** DB에 **+1점**으로 기록합니다.
    * 만약 동일한 조합이 이미 후보 DB에 있다면, 점수를 +1씩 누적합니다.
    * **후보 규칙의 점수가 특정 임계값(예: 10점)에 도달하면**, 우리 시스템 관리자에게 알림을 보냅니다. 관리자는 검토 후, 이 후보를 **신뢰도 70점의 새로운 정식 규칙**으로 `룰 DB`에 추가합니다.
