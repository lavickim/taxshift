# MoneyShift AI 세무서비스 시스템 PRD (Product Requirements Document)

## 시스템 현황 및 구현 상태

### 📊 전체 아키텍처 현황

**핵심 철학**: 다층 방어(Defense-in-Depth) 아키텍처를 통한 비용/속도/지능의 최적 균형

- ✅ **목표 분포**: 룰엔진(L0-L2) 99% + LLM(L3) 1%
- ✅ **현재 성과**: 분류 성공률 **89%** (741% 개선), 11,418개 브랜드 처리

## I. 실시간 거래 처리 파이프라인 - 4계층 시스템

### Layer 0: 캐시 조회 시스템

**처리 비중 목표: 50%+**

- ✅ **Redis 캐시 시스템** 구축 완료
- ✅ **PostgreSQL transaction_cache 테이블** 구현
- ✅ **SHA256 해시 기반** 1ms 미만 조회
- ✅ **캐시 새로고침 API** 구현 (`/v2/tag-mapping-mgmt/refresh-cache`)
- ✅ **어드민 툴 연동** 완료

**구현된 기술 스택:**

- Redis TTL 기반 고성능 캐시
- PostgreSQL 백업 캐시 테이블
- Java Spring Boot 캐시 관리 API

### Layer 1: 정규식/휴리스틱 필터

**처리 비중 목표: 30%+**

- ✅ **Java 백엔드 정규식 엔진** 구현
- ✅ **regex_rules 테이블** 기반 패턴 매칭
- ✅ **80개 키워드 그룹** 생성 및 활성화
- ✅ **한국어 특화 패턴** (카센터, 주유소, 편의점 등)
- ✅ **프랜차이즈 브랜드 인식** 시스템

**주요 성과:**

- 15개 신규 키워드 그룹 추가
- 카센터, 주유소, 냉면, 갈비, 곱창, 피자, 치킨 등 세분화
- 브랜드명 변형 처리 (줄임말, 영어/한글 혼용)

### Layer 2: 유추/일반화 ML

**처리 비중 목표: 4%+**

- ⏳ **계획 수립 완료** - 구현 대기
- ⏳ **Text Classification 모델** 아키텍처 설계
- ⏳ **Sentence-Transformer** 임베딩 기반
- ⏳ **95% 신뢰도 임계값** 정의

**예정 기술 스택:**

- 텍스트 분류 ML 모델
- rules 테이블 학습 데이터 활용
- 유사도 기반 자동 분류

### Layer 3: LLM 정규화 및 심층 분석

**처리 비중 목표: 1% 미만**

- ✅ **Gemini AI 연동** 완료
- ✅ **정규화 프롬프트 v4.0** 구현
- ✅ **심층 분석 프롬프트 v3.1** 활용
- ✅ **비용 최적화** - 1% 미만 호출률 달성
- ✅ **프롬프트 기반 룰 생성** 시스템

## III. Java Spring Boot API 시스템

### ✅ 키워드 관리 API 완전 구현

**엔드포인트 현황:**

1. **`POST /v2/tag-mapping-mgmt/keyword-groups`**
   - ✅ 새로운 키워드 그룹 생성
   - ✅ 동의어 배열 지원
   - ✅ 카테고리별 분류

2. **`GET /v2/tag-mapping-mgmt/keyword-groups`**
   - ✅ 전체 키워드 그룹 조회
   - ✅ 필터링 및 정렬 지원

3. **`POST /v2/tag-mapping-mgmt/refresh-cache`**
   - ✅ 백엔드 캐시 새로고침
   - ✅ 실시간 룰 업데이트 반영

4. **`POST /api/v2/keyword-system/classify`**
   - ✅ 실시간 거래 분류
   - ✅ 4-Layer Processing Pipeline
   - ✅ suggestedTag, suggestedAccount 반환

### ✅ 4-Layer Processing Pipeline 구현

**처리 순서:**

1. **Cache Layer** → Redis/PostgreSQL 캐시 조회
2. **Keyword Engine** → 키워드 그룹 매칭
3. **Dynamic Brand** → 프랜차이즈 브랜드 인식
4. **ML/LLM Layer** → Gemini AI 최종 분류
