# 정규식 패턴 정규화 및 우선순위 시스템 PRD

## 1. 개요

### 1.1 목적
거래 내역 텍스트를 정확하고 효율적으로 분류하기 위한 계층적 정규식 패턴 매칭 시스템을 설계합니다. 브랜드 인식의 정확도를 최우선으로 하며, 사용자 피드백을 통해 지속적으로 개선되는 자가학습 시스템을 구축합니다.

### 1.2 핵심 원칙
- **브랜드 우선**: 브랜드명 매칭이 가장 높은 신뢰도와 우선순위를 가집니다
- **계층적 접근**: 정확도가 높은 패턴부터 순차적으로 적용합니다
- **동적 최적화**: 사용 패턴과 피드백을 통해 시스템이 자동으로 개선됩니다
- **성능 고려**: 빈번한 패턴을 우선 검사하여 처리 속도를 최적화합니다

## 2. 패턴 계층 구조

### 2.1 5단계 계층 시스템

#### Level 1: 브랜드 정확 매칭 (우선순위: 95-100)
- **특징**: 완전히 일치하는 브랜드명
- **신뢰도**: 95% 이상
- **예시**: "스타벅스", "STARBUCKS", "맥도날드", "McDonald's"
- **처리 방식**: 대소문자 무시, 공백 정규화 후 정확한 매칭

#### Level 2: 브랜드 변형 매칭 (우선순위: 85-94)
- **특징**: 브랜드의 약어, 일반적인 변형, 오타
- **신뢰도**: 85-94%
- **예시**: "스벅", "맥날", "GS25편의점", "CU점"
- **처리 방식**: 알려진 변형 패턴 데이터베이스 활용

#### Level 3: 프랜차이즈 패턴 (우선순위: 75-84)
- **특징**: 프랜차이즈 특유의 명명 규칙
- **신뢰도**: 75-84%
- **예시**: "○○점 스타벅스", "맥도날드 ○○DT점"
- **처리 방식**: 위치 정보를 제외한 브랜드명 추출

#### Level 4: 카테고리 키워드 (우선순위: 65-74)
- **특징**: 업종을 나타내는 일반 키워드
- **신뢰도**: 65-74%
- **예시**: "커피전문점", "주유소", "편의점", "마트"
- **처리 방식**: 키워드 기반 카테고리 분류

#### Level 5: 컨텍스트 패턴 (우선순위: 50-64)
- **특징**: 문맥 정보를 활용한 추론
- **신뢰도**: 50-64%
- **예시**: 금액대, 거래 시간, 위치 정보 기반 추론
- **처리 방식**: 복합적인 요소를 고려한 확률적 매칭

### 2.2 우선순위 결정 요소

1. **브랜드 인지도 가중치**
   - 글로벌 브랜드: +5점
   - 전국 체인: +3점
   - 지역 체인: +1점

2. **패턴 정확도 가중치**
   - 정규식 복잡도가 낮을수록 높은 점수
   - 매칭 범위가 좁을수록 높은 점수

3. **사용 빈도 가중치**
   - 최근 30일 매칭 횟수에 따라 동적 조정
   - 자주 사용되는 패턴일수록 우선 검사

## 3. 패턴 정규화 프로세스

### 3.1 전처리 단계

1. **텍스트 정제**
   - 불필요한 특수문자 제거: (주), ㈜, [], () 등
   - 중복 공백 제거 및 트림
   - 일반적인 접미사 제거: "점", "지점", "센터", "매장"

2. **정규화 규칙**
   - 한글↔영문 브랜드명 통일
   - 숫자 표기 통일 (예: "7-ELEVEN" → "세븐일레븐")
   - 알려진 약어 확장 (예: "GS" → "GS칼텍스" in 주유소 컨텍스트)

3. **토큰화**
   - 의미 있는 단위로 텍스트 분리
   - 브랜드명과 부가 정보 구분

### 3.2 매칭 알고리즘

1. **캐스케이드 매칭**
   - Level 1부터 순차적으로 매칭 시도
   - 첫 번째 매칭 시 즉시 반환 (Early Exit)
   - 신뢰도 임계값(80%) 이상일 때만 확정

2. **다중 매칭 처리**
   - 동일 레벨에서 복수 매칭 시 추가 점수 계산
   - 브랜드 인지도, 최근 사용 빈도, 사용자별 선호도 고려

3. **폴백 메커니즘**
   - 모든 레벨에서 매칭 실패 시 LLM 추론으로 전환
   - LLM 결과를 신규 패턴 후보로 저장

## 4. 동적 신뢰도 조정 시스템

### 4.1 피드백 유형별 조정값

1. **긍정적 피드백**
   - 사용자 확인(1클릭): 신뢰도 +2%
   - 반복 사용(5회 이상): 신뢰도 +5%
   - 오류 없는 장기 사용(30일): 신뢰도 +10%

2. **부정적 피드백**
   - 사용자 수정: 신뢰도 -10%
   - 다른 옵션 선택: 신뢰도 -5%
   - 패턴 충돌 발생: 신뢰도 -3%

3. **신뢰도 경계값**
   - 최대: 99% (100%는 시스템 예약)
   - 최소: 10% (그 이하는 패턴 비활성화)
   - 50% 이하: 사용자 확인 필수

### 4.2 학습 메커니즘

1. **패턴 승격/강등**
   - 신뢰도 90% 이상 유지(7일): 상위 레벨로 승격 검토
   - 신뢰도 30% 이하 하락: 하위 레벨로 강등 또는 비활성화

2. **신규 패턴 생성**
   - 동일한 LLM 추론 결과 10회 이상 확인 시
   - 관리자 검토 후 Level 3으로 신규 등록
   - 초기 신뢰도 70%로 시작

## 5. 충돌 해결 전략

### 5.1 충돌 유형

1. **동일 레벨 충돌**
   - 해결: 브랜드 인지도 > 최근 사용 빈도 > 패턴 생성일 순

2. **크로스 레벨 충돌**
   - 해결: 항상 상위 레벨 우선 (Level 1 > Level 2 > ...)

3. **부분 매칭 충돌**
   - 해결: 더 긴 매칭 우선 (최장 일치 원칙)

### 5.2 컨텍스트 기반 해결

1. **사용자 히스토리**
   - 최근 30일 거래 패턴 분석
   - 자주 사용하는 브랜드/카테고리 우선

2. **시간대별 가중치**
   - 점심시간 → 음식점 카테고리 가중치 상승
   - 출퇴근 시간 → 교통 카테고리 가중치 상승

3. **금액대별 추론**
   - 5천원 이하 → 편의점/카페 가능성 높음
   - 3-5만원 → 주유소/마트 가능성 높음

## 6. 성능 최적화 전략

### 6.1 캐싱 전략

1. **메모리 캐시**
   - 최근 1,000개 거래 결과 저장
   - LRU(Least Recently Used) 정책 적용
   - 사용자별 개인화 캐시 분리

2. **패턴 인덱싱**
   - 첫 글자 기반 인덱스 구축
   - 브랜드명 첫 음절로 빠른 필터링

### 6.2 처리 순서 최적화

1. **빈도 기반 정렬**
   - 매일 자정 패턴 사용 통계 집계
   - 사용 빈도 순으로 패턴 재정렬

2. **조건부 스킵**
   - 특정 조건 불만족 시 전체 레벨 스킵
   - 예: 한글만 있으면 영문 브랜드 패턴 스킵

## 7. 모니터링 및 개선

### 7.1 핵심 지표

1. **정확도 지표**
   - 첫 번째 추천 수락률
   - 사용자 수정 비율
   - 평균 신뢰도 점수

2. **성능 지표**
   - 평균 처리 시간
   - 캐시 적중률
   - 일일 처리량

### 7.2 지속적 개선

1. **A/B 테스트**
   - 새로운 패턴의 단계적 배포
   - 알고리즘 변경의 영향도 측정

2. **정기 리뷰**
   - 주간: 신규 패턴 후보 검토
   - 월간: 전체 패턴 성능 분석
   - 분기별: 알고리즘 개선 사항 적용

## 8. 구현 로드맵

### Phase 1: 기반 구축 (1-2개월)
- 5단계 계층 구조 구현
- 기본 전처리 및 정규화 로직
- 초기 브랜드 패턴 DB 구축

### Phase 2: 지능화 (2-3개월)
- 동적 신뢰도 조정 시스템
- 충돌 해결 알고리즘
- 사용자 피드백 수집 체계

### Phase 3: 최적화 (3-4개월)
- 성능 최적화 및 캐싱
- 컨텍스트 기반 추론 고도화
- 모니터링 대시보드 구축

### Phase 4: 확장 (4개월 이후)
- 신규 카테고리 자동 발견
- 지역별/업종별 커스터마이징
- 다국어 지원
