# 📊 데이터 수집 현황 및 전략

*최종 업데이트: 2025년 1월 24일*

## 🎯 전체 전략 개요

MoneyShift AI의 데이터 수집 전략은 다양한 공공데이터와 외부 소스에서 사업체 정보를 수집하여 통합 데이터베이스를 구축하는 것입니다.

### 📈 수집 → 가공 → 통합 파이프라인

```
[다양한 소스] → [임시 수집 테이블] → [데이터 정제/가공] → [통합 DB] → [AI 분석]
```

## 📋 현재 수집된 데이터 현황

- GPS 좌표 (위도/경도)
- 수집 배치 정보, 데이터 소스

## 🔄 데이터 통합 전략

### Phase 1: 수집 단계 (현재)
- ✅ 각 데이터 소스별 임시 테이블에 원본 데이터 저장
- ✅ 수집 배치 추적 및 중복 방지
- ✅ 실시간 수집 모니터링

### Phase 2: 정제 단계 (다음 단계)
- 🔄 사업자등록번호 표준화 및 검증

- 🔄 업종 분류 표준화
- 🔄 중복 사업체 식별 및 병합

### Phase 3: 통합 단계 (향후 계획)
- 📋 마스터 사업체 테이블 구축
- 📋 데이터 품질 점수 산정
- 📋 실시간 동기화 체계 구축

### Phase 4: AI 분석 단계 (최종 목표)
- 📋 4계층 방어 시스템 연계
- 📋 자동 거래 분류 및 세무 처리
- 📋 이상 거래 패턴 탐지

## 📊 수집 성능 지표

### 현재 성능
- **수집 속도**: 초당 약 2-3 레코드 (API 제한 고려)
- **성공률**: 95%+ (중복 제외)
- **오류율**: 5% 미만
- **데이터 품질**: A급 (공공데이터 기반)

### 최적화 계획
- 🔄 병렬 수집 프로세스 구현
- 🔄 캐싱 시스템 도입
- 🔄 증분 업데이트 시스템
- 🔄 자동 오류 복구

## 🛠 기술 스택

### 현재 사용 기술
- **Backend**: Next.js 15 API Routes
- **Database**: PostgreSQL (Supabase)
- **ORM**: Prisma
- **Queue**: 메모리 기반 (향후 Redis 예정)
- **Monitoring**: 실시간 로그 및 토스트 알림

### 향후 도입 예정
- **Message Queue**: Redis/Bull
- **Data Pipeline**: Apache Airflow
- **Data Lake**: MinIO 또는 AWS S3
- **Real-time Analytics**: ClickHouse
