# MoneyShift Launcher & Chrome Extension 통합 시스템 설계

## 개요

MoneyShift AI Platform을 위한 통합 관리 시스템으로, 개발자가 다양한 환경에서 시스템을 쉽게 제어할 수 있도록 하는 두 가지 도구를 제공합니다:

1. **mshift-launcher**: Electron 기반 데스크톱 앱
2. **mshift-chromeext**: Chrome 확장 프로그램

## 핵심 구성 요소

### 1. WebSocket 통신 서버 (포트 9999)
- **역할**: Launcher와 Chrome Extension 간의 실시간 통신 중계
- **기능**:
  - 서비스 상태 실시간 브로드캐스트
  - 명령어 중계 (시작/중지/재시작)
  - 로그 스트리밍
  - 시스템 메트릭스 전달

### 2. mshift-launcher (Electron 앱)
- **역할**: 메인 시스템 관리 도구
- **기능**:
  - 시각적 서비스 제어 패널
  - 실시간 로그 뷰어
  - 시스템 메트릭스 모니터링
  - 자동 업데이트 관리
  - 개발 환경 설정

### 3. mshift-chromeext (Chrome 확장)
- **역할**: 브라우저에서 빠른 접근 및 제어
- **기능**:
  - 팝업 UI로 서비스 상태 확인
  - 원클릭 서비스 제어
  - 개발자 도구 패널 통합
  - 탭별 MoneyShift 프로젝트 감지

## 통신 프로토콜

### WebSocket 메시지 형식
```json
{
  "type": "command|status|log|metrics",
  "source": "launcher|extension",
  "target": "all|launcher|extension",
  "data": {
    // 메시지별 데이터
  },
  "timestamp": "2025-07-27T12:00:00Z",
  "id": "unique-message-id"
}
```

### 주요 메시지 타입

1. **서비스 제어 명령**
```json
{
  "type": "command",
  "source": "extension",
  "data": {
    "action": "start|stop|restart",
    "service": "backend|frontend|all",
    "options": {}
  }
}
```

2. **서비스 상태 업데이트**
```json
{
  "type": "status",
  "source": "launcher",
  "data": {
    "services": {
      "backend": {
        "status": "running|stopped|error",
        "pid": 12345,
        "port": 8080,
        "uptime": "00:15:30"
      }
    }
  }
}
```

3. **로그 스트리밍**
```json
{
  "type": "log",
  "source": "launcher",
  "data": {
    "service": "backend",
    "level": "info|error|warning",
    "message": "Server started successfully",
    "timestamp": "2025-07-27T12:00:00Z"
  }
}
```

## 사용자 시나리오

### 시나리오 1: 데스크톱에서 개발
1. Launcher 앱 실행
2. 시각적 대시보드에서 서비스 상태 확인
3. 원클릭으로 전체 시스템 시작
4. 실시간 로그 모니터링

### 시나리오 2: 웹 개발 중 빠른 제어
1. Chrome에서 MoneyShift 관련 작업 중
2. 확장 아이콘 클릭으로 팝업 열기
3. 서비스 상태 빠른 확인
4. 필요한 서비스만 재시작

### 시나리오 3: 디버깅 중 개발자 도구 활용
1. 브라우저에서 API 테스트 중
2. F12 개발자 도구의 MoneyShift 패널 열기
3. 실시간 백엔드 로그 확인
4. API 응답과 로그 동시 분석

## 보안 고려사항

### Launcher 보안
- 프로세스 격리 (Electron contextIsolation)
- 안전한 IPC 통신
- 로컬 파일 시스템 접근 제한

### Chrome Extension 보안
- Content Security Policy 적용
- 최소 권한 원칙
- 안전한 메시지 패싱

### WebSocket 보안
- 로컬호스트 접근만 허용
- 토큰 기반 인증
- 명령어 검증 및 제한

## 개발 우선순위

### Phase 1 (MVP)
1. WebSocket 서버 구현
2. Launcher 기본 기능 (시작/중지/상태)
3. Chrome Extension 팝업 UI

### Phase 2 (Enhanced)
1. 실시간 로그 스트리밍
2. 개발자 도구 패널
3. 시스템 메트릭스 모니터링

### Phase 3 (Advanced)
1. 자동 업데이트
2. 설정 동기화
3. 알림 시스템

이 설계는 개발자의 워크플로우를 최대한 효율적으로 만들면서도 확장 가능한 아키텍처를 제공합니다.
