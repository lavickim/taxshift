# v1.0 데이터 통합 마스터 플랜

## 📋 프로젝트 개요

**목적**: mshift-data 디렉토리의 원본 데이터를 체계적으로 분석하여 데이터베이스에 통합하고 어드민 UI를 확장

**범위**: 
- 기존 DB 테이블과 중복 데이터 식별
- 새로운 데이터 유형별 테이블 설계  
- 복구 대상 데이터 식별 및 복원
- 어드민 툴 UI 확장
- 디렉토리 구조화

## 🔍 Phase 1: 데이터 분석 및 분류

### 1.1 원본 데이터 목록 분석

mshift-data 디렉토리 내 파일들을 다음 기준으로 분류:

**A. 지역별 음식점 데이터**
- 서울시: `서울시 일반음식점 인허가 정보.csv`, `서울시_일반음식점_utf8.csv`
- 경기도: 부천시, 안산시, 안성시, 하남시
- 강원도: 원주시, 화천군, 정선군  
- 경상북도: 경주시, 구미시, 군위군, 김천시, 문경시, 예천군, 칠곡군, 영덕군
- 광주광역시: 남구, 광산구
- 대구광역시: 동구, 중구
- 부산광역시: 수영구
- 울산광역시: 남구, 중구
- 인천광역시: 남동구, 미추홀구, 부평구, 서구
- 전라남도: 목포시, 순천시, 광양시
- 전북특별자치도: 김제시, 완주군, 임실군
- 충청남도: 공주시, 금산군, 서천군, 청양군
- 충청북도: 괴산군

**B. 국민연금 사업장 데이터**
- `국민연금공단_국민연금 가입 사업장 내역_20250623.csv`

**C. 기타 업종 데이터**
- 주유소: `산업통상자원부_전국 주유소 등록현황_20241231.csv`
- 제조업: `광주광역시_제조업체 현황_20221231.csv`
- 관광: `광주광역시_광주관광명소_20240822.csv`

### 1.2 기존 DB 테이블 매칭

**기존 테이블과 매칭되는 데이터:**
- `datacollection_seoul_restaurants` ← 서울시 음식점 데이터
- `datacollection_gyeonggi_delivery` ← 경기도 배달특급 (이미 복구 완료)

**새로운 테이블이 필요한 데이터:**
- 국민연금 사업장 → `datacollection_national_pension`
- 프랜차이즈 브랜드 → `datacollection_franchise_brands` 
- 지역별 음식점 → `datacollection_regional_restaurants`
- 주유소 → `datacollection_gas_stations`
- 제조업체 → `datacollection_manufacturing`
- 관광명소 → `datacollection_tourism`

## 🏗️ Phase 3: 데이터베이스 스키마 설계

### 3.1 새로운 테이블 스키마

**A. 국민연금 사업장**
```sql
CREATE TABLE datacollection_national_pension (
    id BIGSERIAL PRIMARY KEY,
    data_year_month VARCHAR(6),
    workplace_name TEXT,
    business_registration_number VARCHAR(20),
    workplace_status_code VARCHAR(10),
    postal_code VARCHAR(10),
    address_jibun TEXT,
    address_road TEXT,
    legal_dong_code VARCHAR(10),
    admin_dong_code VARCHAR(10),
    sido_code VARCHAR(5),
    sigungu_code VARCHAR(5),
    eubmyeondong_code VARCHAR(5),
    workplace_type_code VARCHAR(10),
    industry_code VARCHAR(10),
    industry_name VARCHAR(100),
    application_date VARCHAR(8),
    re_registration_date VARCHAR(8),
    withdrawal_date VARCHAR(8),
    member_count INTEGER,
    monthly_notice_amount BIGINT,
    new_acquisition_count INTEGER,
    loss_count INTEGER,
    batch_id VARCHAR(50),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**B. 지역별 음식점 통합**
```sql
CREATE TABLE datacollection_regional_restaurants (
    id BIGSERIAL PRIMARY KEY,
    region_code VARCHAR(10),
    region_name VARCHAR(50),
    license_number VARCHAR(50),
    business_name VARCHAR(200),
    business_type VARCHAR(100),
    business_status VARCHAR(50),
    phone_number VARCHAR(20),
    address_jibun TEXT,
    address_road TEXT,
    postal_code VARCHAR(10),
    x_coordinate DECIMAL(15,10),
    y_coordinate DECIMAL(15,10),
    license_date VARCHAR(10),
    data_source VARCHAR(100),
    batch_id VARCHAR(50),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**C. 프랜차이즈 브랜드**
```sql
CREATE TABLE datacollection_franchise_brands (
    id BIGSERIAL PRIMARY KEY,

    brand_name VARCHAR(200),
    brand_name_english VARCHAR(200),
    priority INTEGER,
    batch_id VARCHAR(50),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```
