# AI 기장 시스템 PRD (Product Requirements Document)

## 1. 제품 개요

### 1.1 비전
은행 거래 내역을 자동으로 분석하여 태그를 부여하고, 회계 계정과목으로 매핑하여 자동으로 재무제표를 생성하는 AI 기반 기장 시스템

### 1.2 핵심 가치
- **자동화**: 거래 내역 분류 및 회계 처리 자동화
- **학습**: 사용자 피드백 기반 지속적 개선
- **정확성**: 하이브리드 룰 엔진과 AI의 조합으로 높은 정확도 달성
- **편의성**: 복잡한 회계 지식 없이도 재무제표 생성 가능

## 2. 시스템 아키텍처

### 2.1 핵심 구성요소

#### 2.1.1 하이브리드 룰 엔진
- **룰 DB 우선 매칭**: 검증된 규칙 기반 빠른 처리
- **LLM 폴백**: 새로운 패턴에 대한 AI 추론
- **동적 학습**: 사용자 피드백 기반 신뢰도 조정

#### 2.1.2 정규식 처리 엔진
- **거래처명 정규화**: 200개 이상의 정규식 패턴
- **카테고리 자동 분류**: 100개 세부 카테고리
- **개인정보 마스킹**: 보안 강화

#### 2.1.3 회계 처리 엔진
- **계정과목 매핑**: 태그 → 회계 계정과목 자동 변환
- **복식부기 처리**: 차변/대변 자동 생성
- **재무제표 생성**: 대차대조표, 손익계산서 자동 생성

### 2.2 데이터 흐름

```
[거래 내역 입력] 
    ↓
[정규식 전처리] → 거래처명 정규화, 개인정보 마스킹
    ↓
[하이브리드 룰 엔진]
    ├→ [룰 DB 검색] → 매칭 성공 → [신뢰도 확인]
    └→ [LLM 추론] → 새로운 패턴 처리
    ↓
[사용자 확인/수정]
    ↓
[회계 계정과목 매핑]
    ↓
[재무제표 생성]
```

## 3. 기능 요구사항

### 3.1 거래 내역 처리

#### 3.1.1 자동 분류
- **입력**: 은행 거래 내역 (거래처명, 금액, 날짜)
- **처리**: 
  - 정규식 기반 거래처명 정규화
  - 카테고리 자동 분류 (100개 카테고리)
  - 태그 자동 부여
- **출력**: 분류된 거래 내역

#### 3.1.2 사용자 상호작용
- **신뢰도 높음**: 자동 분류 후 확인만 요청
- **신뢰도 중간**: 질문을 통한 추가 정보 수집
- **신뢰도 낮음**: 사용자 직접 선택/수정

### 3.2 회계 처리

#### 3.2.1 계정과목 매핑 테이블

| 카테고리 | 태그 예시 | 차변 계정과목 | 대변 계정과목 |
|---------|----------|-------------|-------------|
| 커피전문점 | #커피, #카페 | 복리후생비 | 현금/예금 |
| 음식점 | #식사, #회식 | 접대비/복리후생비 | 현금/예금 |
| 주유소 | #주유, #교통 | 차량유지비 | 현금/예금 |
| 병원 | #의료, #건강 | 의료비 | 현금/예금 |
| 통신요금 | #통신, #휴대폰 | 통신비 | 현금/예금 |
| 사무용품 | #문구, #사무 | 소모품비 | 현금/예금 |
| 교육 | #교육, #학원 | 교육훈련비 | 현금/예금 |
| 보험료 | #보험 | 보험료 | 현금/예금 |
| 임대료 | #임대, #월세 | 임차료 | 현금/예금 |
| 급여 | #급여 | 인건비 | 현금/예금 |

#### 3.2.2 복식부기 처리
- 모든 거래에 대해 차변/대변 자동 생성
- 거래 유형에 따른 계정과목 자동 선택
- 부가세 자동 계산 및 분리 (해당하는 경우)

### 3.4 학습 및 개선

#### 3.4.1 신뢰도 업데이트 로직
- **긍정 피드백**: 신뢰도 +1점 (최대 100점)
- **부정 피드백**: 신뢰도 -2점 (최소 0점)
- **순위 재조정**: 사용자 선택 기반 자동 조정

#### 3.4.2 신규 규칙 생성
- LLM 추론 결과 누적 (임계값: 10회)
- 관리자 검토 후 정식 규칙 등록
- 초기 신뢰도 70점으로 시작

## 4. 비기능 요구사항

### 4.1 성능
- 거래 내역 처리: 1,000건/초 이상
- 응답 시간: 평균 100ms 이내
- 동시 사용자: 10,000명 이상 지원

### 4.2 확장성
- 마이크로서비스 아키텍처
- 수평적 확장 가능
- 클라우드 네이티브 설계

### 4.3 보안
- 개인정보 암호화 저장
- 거래 내역 마스킹 처리
- 접근 권한 관리

### 4.4 사용성
- 직관적인 UI/UX
- 모바일 지원
- 다국어 지원 (한국어, 영어)

## 5. 기술 스택 권장사항

### 5.1 백엔드
- **Java + Spring Boot** (대규모 트래픽 처리)
  - 안정적인 성능
  - 우수한 멀티스레딩
  - 엔터프라이즈 지원

### 5.2 데이터베이스
- **PostgreSQL**: 메인 트랜잭션 DB
- **Redis**: 캐싱 및 세션 관리
- **Elasticsearch**: 거래 내역 검색

### 5.3 AI/ML
- **LLM API**: OpenAI 또는 Claude API
- **규칙 엔진**: Drools 또는 자체 구현


## 6. 구현 단계

### Phase 1: MVP (3개월)
- 기본 거래 내역 분류
- 핵심 카테고리 30개
- 기본 재무제표 생성

### Phase 2: 확장 (3개월)
- 전체 100개 카테고리 지원
- 학습 엔진 고도화
- 상세 재무분석 기능

### Phase 3: 고도화 (3개월)
- 예측 분석 기능
- 세무 신고 지원
- API 제공

## 7. 성공 지표

- 자동 분류 정확도: 95% 이상
- 사용자 수정률: 5% 이하
- 처리 시간: 100ms 이내

### 7.2 정성적 지표
- 사용자 만족도: 4.5/5.0 이상
- 회계 업무 시간 단축: 80% 이상
- 재무제표 정확도: 99% 이상
